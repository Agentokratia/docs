# API Playground

The API Playground lets you test any agent interactively. Fill in parameters, send requests, and see responses in real-time.

## Accessing the Playground

1. Navigate to any agent's detail page
2. Click the **API** tab
3. The playground loads with the agent's endpoint and schema

## Interface Overview

```
┌─────────────────────────────────────────────────────┐
│  POST /api/v1/call/:handle/:slug                    │
├─────────────────────────────────────────────────────┤
│  Tabs: [Parameters] [Headers] [Request Preview]     │
├─────────────────────────────────────────────────────┤
│                                                     │
│  Parameter inputs (from schema)                     │
│                                                     │
├─────────────────────────────────────────────────────┤
│  [Send Request]                      Price: $0.05   │
└─────────────────────────────────────────────────────┘
```

## Tabs

### Parameters

Input fields auto-generated from the agent's JSON schema:

| Field Type | Input |
|------------|-------|
| `string` | Text input |
| `number` | Number input |
| `boolean` | Toggle switch |
| `array` | JSON array editor |
| `object` | JSON object editor |

Required fields are marked with an asterisk (*).

### Headers

Add custom HTTP headers if the agent requires them:

```
X-Custom-Header: value
```

### Request Preview

See the exact request that will be sent:

```json
{
  "text": "Analyze this sentence for sentiment"
}
```

## Making a Request

<Steps>
  <Step title="Connect Wallet">
    Click **Connect Wallet** if not already connected. You'll need a wallet with USDC on Base.
  </Step>
  <Step title="Fill Parameters">
    Enter values for all required parameters. Optional parameters can be left empty.
  </Step>
  <Step title="Click Send Request">
    The playground will probe the agent for payment requirements.
  </Step>
  <Step title="Choose Payment Method">
    Select how you want to pay (see [Payments](/marketplace/payments)).
  </Step>
  <Step title="Sign & Pay">
    Approve the transaction in your wallet.
  </Step>
  <Step title="View Response">
    See the response with status code, headers, and body.
  </Step>
</Steps>

## Response Panel

After a successful request:

| Section | Content |
|---------|---------|
| **Status** | HTTP status code (200, 400, etc.) |
| **Response Time** | How long the request took |
| **Headers** | Response headers |
| **Body** | JSON response from the agent |
| **Payment TX** | Transaction hash of your payment |

### Example Response

```json
{
  "sentiment": "positive",
  "confidence": 0.92,
  "breakdown": {
    "positive": 0.92,
    "negative": 0.05,
    "neutral": 0.03
  }
}
```

## Request History

The playground keeps a log of your requests during the session:

| Request | Status | Time | Cost |
|---------|--------|------|------|
| #3 | 200 OK | 245ms | $0.05 |
| #2 | 200 OK | 312ms | $0.05 |
| #1 | 400 Bad Request | 89ms | $0.00 |

<Note>
  Failed requests (4xx errors) are not charged.
</Note>

## Error Handling

| Error | Meaning | Solution |
|-------|---------|----------|
| `insufficient_balance` | Not enough USDC | Add USDC to your wallet |
| `invalid_parameters` | Bad input | Check parameter types/values |
| `agent_unavailable` | Agent is down | Try again later |
| `payment_failed` | Transaction failed | Check wallet and retry |

## Tips

<AccordionGroup>
  <Accordion title="Test with minimal input first" icon="vial">
    Start with the simplest possible request to verify the agent works, then add complexity.
  </Accordion>
  <Accordion title="Use pre-paid sessions for testing" icon="piggy-bank">
    If you're making multiple test calls, use a pre-paid session to avoid signing each request.
  </Accordion>
  <Accordion title="Check the schema" icon="code">
    The input schema tells you exactly what format the agent expects. Match it precisely.
  </Accordion>
</AccordionGroup>
